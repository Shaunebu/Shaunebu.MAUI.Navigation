# VSIX Features

## Overview

The **Navigation Session Inspector** tool window exposes four panels, each backed by a dedicated ViewModel from the `Shaunebu.MAUI.Navigation.Vsix.Desktop` assembly. This page documents every panel, its controls, and the underlying data it surfaces.

---

## Opening the Tool Window

**View → Other Windows → Navigation Session Inspector**

The tool window docks like any standard Visual Studio panel. It can be floated, tabbed, or pinned to a side column. State is persisted across Visual Studio sessions.

---

## Panel: Live Events

The **Live Events** panel streams real-time navigation events from a running MAUI debug session that has `UseNavigationDebugger` registered.

| Control | Description |
|---|---|
| **Clear** | Clears the current event list without ending the session |
| **Pause / Resume** | Pauses or resumes live event streaming |
| **Filter** | Filter events by operation type (Push, Pop, Modal, Reset, Flow) |
| **Auto-scroll** | Toggle auto-scrolling to the latest event |

Each event row shows:

| Column | Description |
|---|---|
| Time | Wall-clock timestamp of the operation |
| Type | `Push` / `Pop` / `Modal` / `Reset` / `Flow` |
| Target | Page type or flow name |
| Route | Resolved route string, or `(none)` for flow transitions |
| Duration | Time from operation start to `NavigationResult` returned |
| Result | `✓ Success` or `✗ {FailureReason}` |

Double-click any row to open the **Frame Inspector** for that operation.

---

## Panel: Navigation Stack

The **Navigation Stack** panel shows the live navigation and modal stacks as of the most recently completed operation.

```
Navigation Stack          Modal Stack
─────────────────         ───────────────
[3] SettingsPage          [1] PrivacyPage
[2] HomePage
[1] (Shell root)
```

| Control | Description |
|---|---|
| **Refresh** | Manually refresh stack state (auto-refreshes on each event) |
| **Copy** | Copy the current stack as text |

---

## Panel: Timeline

The **Timeline** panel provides frame-by-frame replay of a recorded or imported session. It does **not** re-execute navigation — it reconstructs navigation history from recorded events.

### Loading a Session

| Source | How |
|---|---|
| Live session | Click **Use Current Session** to load the session from the attached debug process |
| File | Click **Open Session…** and select an exported `.json` file |

### Timeline Controls

| Control | Action |
|---|---|
| **⏮ First** | Jump to the first operation |
| **◀ Back** | Step backward one operation |
| **▶ Forward** | Step forward one operation |
| **⏭ Last** | Jump to the final operation |
| **Seek slider** | Drag to any position in the session |
| **Group by Flow** | Toggle grouping by named flow (Auth, Main, etc.) |

### Frame Detail

Each frame shows:

| Property | Description |
|---|---|
| Operation | Human-readable summary (e.g. `Push → HomePage`) |
| Route | Resolved route at the time of the operation |
| Navigation Stack | Stack snapshot **after** this operation |
| Modal Stack | Modal stack snapshot **after** this operation |
| Flow | Active flow at this point |
| Warnings | Any warnings generated by this operation |
| Stack Diff | `+Added`, `-Removed` pages compared to the previous frame |
| Elapsed | Duration of the operation |

---

## Panel: Warnings

The **Warnings** panel surfaces runtime warnings emitted by `NavigationRuntimeWarningEngine` during the session.

### Warning Severity

| Icon | Severity | Examples |
|---|---|---|
| 🔵 Info | Informational | Route resolved via fallback, flow changed |
| 🟡 Warning | Actionable concern | Guard redirect loop, ambiguous route, unawaited navigation |
| 🔴 Error | Operation failed | Guard rejected, route not found, adapter threw |

### Columns

| Column | Description |
|---|---|
| Time | When the warning was raised |
| Code | Warning code (e.g. `NAV-W004`) |
| Message | Human-readable warning description |
| Operation Id | Correlates the warning to a specific operation in the Timeline panel |

Click **Jump to Operation** to open the associated Timeline frame for a selected warning.

## Exporting From the Tool Window

Click **Export Session** in the Timeline toolbar to save the loaded session as:

- **JSON** — machine-readable for re-import or tooling
- **Markdown** — human-readable report suitable for sharing with teammates

See [Exporting Sessions](../debugger/exporting-sessions.md) for the programmatic export API.

---

## Opening a Session File

To perform a post-mortem review without a running debug session:

1. Export the session file from your device or emulator.
2. Transfer the `.json` file to your development machine.
3. Open the Navigation Session Inspector.
4. Click **Open Session…** and select the `.json` file.
5. Use the Timeline controls to replay each operation.

---

## Keyboard Shortcuts

| Shortcut | Action |
|---|---|
| `F5` | Step forward one frame (Timeline panel focused) |
| `Shift+F5` | Step backward one frame |
| `Home` | Jump to first frame |
| `End` | Jump to last frame |
| `Ctrl+E` | Export current session |
| `Ctrl+O` | Open session file |
